Frequently Asked Questions

1. What the difference between var, let, and const?
'var' declares a variable with function scope and can be re-assigned, 'let' declares a block-scoped variable that can also be re-assigned, while 'const' declares a block-scoped variable that cannot be re-assigned after initialization.
2. What the difference between map(), forEach(), and filter()?
'map()' creates a new array with the results of calling a function on every element, 'forEach()' executes a function on each element without returning anything, and 'filter()' creates a new array with all elements that pass the test implemented by the provided function.
3. Explain arrow functions and how they are different from regular functions
Arrow functions are a shorter way to write functions, and they always use the 'this' value from where they are defined(don't have their own list of arguments), unlike regular functions that get their 'this' based on how they're called.
4. How JavaScript Promises work?
JavaScript Promises represent a value that may not be available yet but will be resolved in the future, allowing you to handle asynchronous operations with then(), catch(), and finally() methods for success, failure, and completion respectively.
5. How closures work in JavaScript?
Closures in JavaScript allow a function to remember and access variables from its lexical scope, even after the outer function has finished executing.
